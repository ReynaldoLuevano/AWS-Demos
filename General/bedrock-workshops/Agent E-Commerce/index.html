Create an E-Commerce Bot

En este laboratorio aprenderás a crear un Agente de recomendación de productos en Amazon Bedrock, conectando Lambdas como Action Groups para interactuar con un catálogo de productos en DynamoDB y gestionar un carrito de compra.

1

Tarea 1: Examina el catálogo de Productos

⚠ Verificación importante
Asegurate que estas en la región N. Virgina - US-EAST-1 antes de comenzar con los pasos
1
Abre la consola de AWS y en la barra de búsqueda de servicios introduce DynamoDB. En los resultados de la búsqueda, en la sección Servicios, haz clic en DynamoDB.
2
En la consola de DynamoDB, haz clic en Tables en el menú izquierdo y a continuación haz clic en la tabla con nombre workshop-Products-XXX.
3
Una vez se carguen los detalles de la tabla, haz clic en Explore Table Items.
4
Asegúrate de que está seleccionada la opción Scan en el apartado Scan or query Items y a continuación haz clic en Run.
5
En el apartado inferior podrás ver todos los elementos de la tabla Productos. Verás que cada uno de ellos tiene una categoría, una ocasión en donde se puede regalar el producto, para qué género está orientado el producto y una descripción.
2

Tarea 2: Examina las Funciones Lambda

6
En la barra de búsqueda de servicios introduce Lambda. En los resultados de la búsqueda, en la sección Servicios, haz clic en Lambda.
7
Verás en la página de Functions que existen funciones Lambda creadas. Las que vamos a utilizar en este workshop son sólo 3, cada una de ellas realiza una funcionalidad:

Añadir items al carrito de compra (workshop-AddToCartFunction)

Obtener el estado del carrito (workshop-getCartFunction)

Obtener detalles del producto (workshop-getProductDetailsFunction)
8
Si lo deseas, puedes acceder a cualquiera de las funciones para ver el código. Para ello haz clic en el nombre de la función; esto te llevará a la página de detalle donde podrás consultar el código fuente.

Tarea 3: Creando el Agente

9
En el cuadro de búsqueda de la barra de navegación superior, escribe bedrock. En los resultados de la búsqueda, en la sección Servicios, haz clic en Amazon Bedrock.
10
En el panel de navegación izquierdo, desplázate hacia abajo hasta "Build" y haz clic en "Agents".
11
Haz clic en Create Agent.
12
En la ventana emergente, coloca como nombre de agente e-commerce-agent y la siguiente descripción: "Product recommendations agent". Haz clic en Create.
13
Aparece la vista "Agent Builder", donde configuraremos el modelo.
14
En el apartado "Agent Resource Role", selecciona "Use an existing service Role" y del desplegable selecciona el único valor disponible workshop-IAMRolexxx.
15
Haz clic en Select Model.
16
En la ventana emergente, desmarca el check "Bedrock Agents Optimized" ubicado en la parte superior izquierda de dicha ventana emergente.
17
Selecciona Anthropic Claude Haiku 4.5. Asegúrate de que en el apartado Inference está seleccionado US Anthropic Claude Haiku 4.5 y haz clic en Apply.
18
En el apartado Instructions for Agent, completa con lo siguiente:

"You are a Product recommendations agent for gift products, the user is trying to buy a gift for someone and you are trying to help identify the best products based on the filters in the action groups, ask questions to identify at least one of the input filters, gender, category or ocassion. Do not recommend any products that are not retrieved from the products API. Do not ask about the gender if it is obvious from the user input already. Always start by getting the full list of products from the API so you can know the proper filter values to be used in the API parameters. Always use a single value for each filter field, and adthere to the filtration values based on the first API call. And never tell the user about the API and its details"
⚠ Importante
Este conjunto de instrucciones son importantes ya que guían al Agente en cómo comunicarse con el Usuario y en cómo utilizar las APIs.
19
Haz clic en Additional Settings y en el apartado "User Input" selecciona la opción Enabled.
20
⚠️ Importante. Haz clic en Save ubicado en la parte superior antes de continuar con el siguiente paso.
21
En el apartado Action Groups, haz clic en Add para añadir un Action Group que permitirá al Agente invocar la Lambda Function.
22
En el Action Group name, coloca get_product_recommendations y selecciona Define with API Schemas.
23
En el apartado Action Group Invocation, selecciona "Select an existing Lambda Function" y en el desplegable selecciona la Lambda que tiene el nombre workshop-GetProductDetailsFunction.
24
En el apartado Action Group Schema, selecciona "Define via in-line schema editor".
25
Borra el contenido de la sección In-line OpenAPI Schema y copia/pega el siguiente Schema:
OpenAPI Schema – Get Product Recommendations 
openapi: 3.0.0
info:
  title: Product Recommendations API
  description: API for retrieving product recommendations based on filters
  version: 1.0.0

paths:
  /products:
    get:
      summary: Retrieve Product Details
      description: This API retrieves product Information. Filtering parameters are passed as query strings. If query strings are empty it performs a full scan
      operationId: getProducts
      parameters:
        - name: product_name
          in: query
          description: Retrieve details for a specific product name
          schema:
            type: string
        - name: category
          in: query
          description: Filters product by category
          schema:
            type: string
        - name: gender
          in: query
          description: Filters product by gender
          schema:
            type: string
        - name: occasion
          in: query
          required: false
          description: Filters product by ocassion
          schema:
            type: string
      responses:
        '200':
          description: A list of recommended products
          content:
            application/json:
              schema:
                type: array
                items:
                  type: object
                  properties:
                    product_name:
                      type: string
                    category:
                      type: string
                    gender:
                      type: string
                    product_description:
                      type: string
        '400':
          description: Bad request - invalid parameters
        '500':
          description: Internal server error
26
Haz clic en Create.
27
Regresaras a la pantalla Agent Builder. Haz clic en Save And Exit ubicado en la parte superior

Tarea 3: Prueba el Agente

28
En la pantalla donde ves los detalles del Agente, haz clic en Test. En la parte derecha se despliega el Chat para probar el Agente. Verás un mensaje que indica "Prepare the agent to test the latest Changes"..Haz clic en Prepare Prepare
29
Verás un mensaje en verde que indica"Agent: e-commerce was successfully prepared. En la parte inferior derecha, Introduce el siguiente texto: Quiero comprar un regalo y haz clic en Run.
30
El agente te contestará con un mensaje similar a ¿Para quién es el regalo? ¿Es para una mujer, hombre, o alguien sin preferencia específica (unisex)?. Introduce el siguiente texto: Es para mi hermana.
31
El agente te contestará con un mensaje similar a ¿Para qué ocasión es el regalo? Por ejemplo: cumpleaños, Día de la Madre, Día de San Valentín, boda, cita, fiesta, o simplemente un regalo para demostrar aprecio.. Introduce el siguiente texto: Es para su cumpleaños.
32
El agente te contestará con varias opciones. Estas opciones se han recuperado de la tabla de DynamoDB. Haz clic en Show Trace y se desplegará del lado derecho una sección para ver las trazas.
33
Haz clic en la pestaña "Orchestration and Knowledge Base". Ve el detalle de cada traza. En ellas podrás ver que realiza la llamada a la API para obtener los productos.

Borra el contenido de la sección In-line OpenAPI Schema y copia/pega el siguiente Schema:
OpenAPI Schema – Add to Cart 
openapi: 3.0.0
info:
  title: Add To Cart API
  description: This API add a product to a user's cart
  version: 1.0.0
paths:
  /cart:
    post:
      summary: Add a product to the cart
      description: Adds a specified product to the user's shopping cart
      operationId: addToCart
      parameters:
        - name: userId
          in: query
          description: The id of the user
          required: true
          schema:
            type: string
        - name: productName
          in: query
          description: The name of the product to add to the cart
          required: true
          schema:
            type: string
      responses:
        '200':
          description: Product successfully added to cart
          content:
            application/json:
              schema:
                type: string
        '400':
          description: Bad request - missing required parameters
        '500':
          description: Internal server error

Challenge: Completa las funcionalidades al Agente

ℹ️ Información
Nuestro Agente está incompleto. Es necesario que si el usuario pregunta "¿Qué tengo en el carrito?" el agente pueda responder. Para ello, realiza las siguientes tareas. Si tienes dudas, pregunta al instructor.
62
Modifica las instrucciones del Agente para que sepa que puede obtener el estado del carrito si el usuario lo pregunta.
63
Utiliza la función Lambda workshop-getCartFunction para obtener el estado del carrito. Añade un nuevo Action Group con su correspondiente OpenAPI Schema. Si lo necesitas, el esquema se encuentra abajo del todo de esta página (después de Conclusiones)
64
Si has realizado todos los cambios necesarios, prueba el agente con el siguiente diálogo:

Usuario: Quiero un regalo para el cumpleaños de mi padre.
Agente: Tengo estas opciones…
Usuario: OK, me quedo con la primera opción, añádela al carrito, mi usuario es usuario2@amazon.com
Agente: Añadida al carrito
Usuario: Ahora quiero un regalo para el cumpleaños de mi abuela.
Usuario: OK, me quedo con la primera opción, añádela al carrito
Agente: Añadida al carrito

Elimina la sesion con el agente. haciendo clic en el icono localizado en la parte superior del chat

Vuelve a empezar un chat:
Usuario: Muéstrame qué tengo en el carrito mi usuario es usuario2@amazon.com
Agente: Este es el contenido del carrito…
✅ ¡Felicidades! Has completado el challenge y el agente ahora es capaz de mostrar el contenido del carrito al usuario.
🎓 Conclusiones

En este laboratorio, has aprendido a crear un Agente en Amazon Bedrock. Conectaste el agente con funciones Lambda mediante Action Groups y esquemas OpenAPI, integraste un catálogo de productos en DynamoDB y construiste un flujo conversacional completo que incluye recomendaciones personalizadas de regalos y gestión de un carrito de compra.
